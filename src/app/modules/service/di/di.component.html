The counter increments by one in the constructor of components that utilize these services.

The root service is a singleton and increments only by 1 as it's used solely within one component. If provided in another module or directly in a component, it utilizes the same instance.

The module service, also a singleton, increments by 1 each time the primary component in the module is visited and by 1 again when we explicitly access the secondary component. If the service is provided in another module, it creates a new instance.

The component service's lifespan aligns with its host component, which means that the counter never exceeds 1.
